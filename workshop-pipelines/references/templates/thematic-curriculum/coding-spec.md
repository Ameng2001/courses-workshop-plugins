# 主题式课程编码规范

## 编码格式

```text
TC-{category}-{week}-{seq}
```

示例：

- `TC-TA-W1-01`
- `TC-RA-W1-03`
- `TC-OG-W2-01`
- `TC-LR-W3-02`
- `TC-HS-W4-01`

## 活动类型缩写

| 缩写 | 类型 |
|------|------|
| `TA` | Teaching Activity / 教学活动 |
| `RA` | Region Activity / 区域活动 |
| `OG` | Outdoor Game / 户外游戏 |
| `LR` | Life Routine / 生活渗透 |
| `HS` | Home-School / 家园互动 |

## 使用规则

1. 每个活动必须唯一编码。
2. 月矩阵、周编排、活动稿应使用同一编码。
3. 客户自有发展目标编码应作为并列字段记录，不替代平台活动编码。

## 双编码模型

主题式课程项目中建议同时保留两套编码：

### A. 平台活动编码

用于串联：
- 月矩阵
- 周活动安排
- 单活动稿

示例：
- `TC-TA-W2-01`
- `TC-RA-W2-01`

### B. 客户发展目标编码

用于保留客户自定义的目标体系：
- `SE-4`
- `SE-8`

推荐写法：

```yaml
activity_code: TC-TA-W2-01
local_goal_system: HEPU-SE
goal_codes:
  - SE-4
  - SE-8
```

## 输出约束

1. 活动编码必须出现在月矩阵、周安排和活动稿中。
2. 本地目标编码优先出现在教学活动稿中；其他活动类型如有要求也可出现。
3. 如客户未提供完整编码说明，允许先保留编码与简短说明，不强制生成完整映射表。
