# Persona: 王老师（一线带班教师）⭐ 主要用户

> "我每天要备 2-3 节课，光从小红书找素材就要花一个多小时，找到了还不一定能用。"

┌─────────────────────────────────────────────────────────┐
│  王老师                                                  │
│  "给我一个从学期到每天都能用的备课系统，                   │
│   不要让我从零开始拼凑"                                   │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  Role:        带班教师（PreK-4 / 中班）                   │
│  Age range:   23-35                                     │
│  Tech level:  中（熟练使用手机 App，用过豆包/文心等 AI）   │
│  Context:     独立带 20-25 个孩子，配 1 名保育员           │
│               每天有 30 分钟集体教学 + 30 分钟区角活动      │
│               每周需提交周计划，每月需提交月反思             │
│               使用区编教材（如《学习活动》《生活活动》）     │
│                                                         │
│  Goals:                                                 │
│  1. 体系化备课：学期→月→周→日有清晰框架，不再碎片拼凑    │
│  2. 理论落地：把《指南》和教育理论转化为可直接操作的活动    │
│  3. 高效产出：30 分钟内完成一节课的完整教案                │
│  4. 格式规范：教案/期末材料一次成型，不反复调格式          │
│                                                         │
│  Pain points:                                           │
│  1. 教材适配性差：区编教材只给方向不给细节，学期→月→周     │
│     缺乏体系化衔接，每周都在从零开始                      │
│     — severity: 高                                       │
│  2. 理论转化困难：知道皮亚杰/瑞吉欧/4C 但不知道怎么       │
│     转化为具体活动步骤和提问话术                          │
│     — severity: 高                                       │
│  3. 备课低效：依赖小红书/抖音搜碎片素材，找到的内容       │
│     质量参差不齐，多媒体课件（PPT/图片/视频）制作耗时      │
│     — severity: 高                                       │
│  4. 通用大模型不好用：用豆包/文心生成的教案内容空洞，      │
│     目标不清晰，不符合幼教场景的年龄适切性要求             │
│     — severity: 中                                       │
│  5. 格式反复调整：教案格式不统一，期末材料（个人总结/      │
│     班级总结/教研记录）每次都要手动调格式                  │
│     — severity: 中                                       │
│                                                         │
│  Current tools:                                         │
│  - 区编教材（纸质 + 电子版）                              │
│  - 小红书/抖音搜索教学灵感和素材                          │
│  - Word/WPS 手写教案                                     │
│  - 豆包/文心一言等通用 AI（偶尔使用，满意度低）            │
│  - 打印的 PBL 预案 PDF（由课研主任提供）                   │
│  - PBL Box（统一配送的材料盒）                            │
│                                                         │
│  Needs (prioritized):                                   │
│  P0 — 专属知识库：区编教材 + 园本理念 + 历年优秀教案 +    │
│        教研记录 + 资源库，结构化可检索                     │
│  P0 — 课程编排：支持"导入→探究→总结→延伸"四/五步模板，   │
│        卡片式选择活动环节，不局限于 PBL 一种方法论         │
│  P0 — 资源调用与智能生成：图片/视频/绘本推荐/作业纸/      │
│        探究图表的智能匹配和生成                           │
│  P1 — 教案格式智能更正：基于标准模板自动调整教案格式      │
│  P1 — 文件格式自动纠正：期末材料基于模板格式修正          │
│                                                         │
│  Success looks like:                                    │
│  "开学前用 AI 拉好整学期的主题日历，每个月自动拆成         │
│   周计划，每天备课只要选模板、填主题、调一调就能出一份     │
│   格式标准、理论扎实、资源齐全的教案"                     │
│                                                         │
└─────────────────────────────────────────────────────────┘
