# Process Flow: 月度 PBL 预案研发流程

> Process name: 月度 PBL 预案研发
> Trigger: 学期课程日历分配月度主题
> End states: 预案通过审批并发放给教师 / 预案被打回重做
> Boundary: 从主题确定到预案定稿（不含教师执行和课后反思）
> Frequency: 每月 1 次，产出 4-6 套预案（按年龄段）

## Flow Diagram

```
[Trigger: 月度主题已分配]
    │
    ▼
[课研主任查阅课标和往年参考]
    │
    ▼
[撰写项目概览（主题背景 + 教育价值）]
    │
    ▼
[评估儿童先前经验]
    │
    ▼
[映射 4C 能力 + 制定学习目标]
    │
    ▼
[设计驱动性问题]
    │
    ▼
◇ 驱动问题是否足够开放且可探究？
    ├─ 否 → [修改驱动问题] → (回到 ◇)
    │
    └─ 是 ↓
        [构建主题网络图]
            │
            ▼
        [拆分 3 条递进探究线索]
            │
            ▼
        ◇ 3 条线索是否递进且均衡？
            ├─ 否 → [调整线索划分] → (回到 ◇)
            │
            └─ 是 ↓
                [为每条线索设计活动序列]
                    │  ┌─────────────────────┐
                    │  │ 对每个活动:          │
                    │  │ - 确定关键问题       │
                    │  │ - 编写活动内容步骤   │
                    │  │ - 分配活动编码       │
                    │  │   (PBL-C{x}-{y})    │
                    │  │ - 编写教师提示(Tips) │
                    │  └─────────────────────┘
                    ▼
                [匹配资源清单]
                    │  ┌─────────────────────┐
                    │  │ 分类:               │
                    │  │ - PBL Box（统一配送）│
                    │  │ - 探索足迹袋（Journal）│
                    │  │ - 自备材料           │
                    │  │ - 多媒体资源         │
                    │  └─────────────────────┘
                    ▼
                ◇ 资源清单是否完整且可获取？
                    ├─ 否 → [补充/替换资源] → (回到 ◇)
                    │
                    └─ 是 ↓
                        [编写双语内容]
                            │
                            ▼
                        [组装预案文档]
                            │
                            ▼
                        [内部评审（课研组互审）]
                            │
                            ▼
                        ◇ 评审是否通过？
                            ├─ 否 → [根据反馈修改] → (回到评审)
                            │
                            └─ 是 ↓
                                [提交园长审批]
                                    │
                                    ▼
                                ◇ 园长是否批准？
                                    ├─ 否 → [根据批注修改] → (回到园长审批)
                                    │
                                    └─ 是 ↓
                                        [发放给带班教师 + 下单 PBL Box]
                                            │
                                            ▼
                                        [END: 预案已交付]
```

## Decision Points

| ID | Decision | Condition | True path | False path |
|----|----------|-----------|-----------|------------|
| D1 | 驱动问题是否足够开放且可探究？ | 问题允许多种答案 + 儿童可通过动手探索回答 + 与月度主题相关 | 构建网络图 | 修改驱动问题（可能循环 3-5 次） |
| D2 | 3 条线索是否递进且均衡？ | 每条线索 3-5 天 + 线索间有逻辑递进 + 覆盖驱动问题不同维度 | 设计活动 | 调整线索划分 |
| D3 | 资源清单是否完整且可获取？ | 每个活动都有资源 + 分类明确 + PBL Box 可供应 | 编写双语 | 补充/替换资源 |
| D4 | 评审是否通过？ | 课标对齐 + 活动可执行 + 资源齐全 + 双语一致 | 提交园长 | 修改（可能 1-3 轮） |
| D5 | 园长是否批准？ | 整体质量合格 + 格式规范 + 无安全隐患 | 发放交付 | 修改（通常 1 轮） |

## Actor Swim Lanes

```
课研主任:  [查课标] → [写概览] → [评估经验] → [映射4C] → [设计问题] → [画网络图]
                → [拆线索] → [设计活动] → [匹配资源] → [写双语] → [组装文档] ──┐
                                                                              │
课研组:                                                              [互审] ──┤
                                                                              │
园长:                                                                [审批] ──┤
                                                                              │
供应链:                                                              [PBL Box 下单] ─→ [配送]
                                                                              │
教师:                                                                [接收预案] ─→ [执行]
```

## Key Observations

1. **D1（驱动问题验证）是最大瓶颈** — 平均循环 3-5 次，每次修改都可能影响下游网络图和线索设计
2. **活动设计到资源匹配是并行可能区** — 不同线索的活动可以并行设计，资源匹配也可以在活动设计中同步进行
3. **双语内容是独立工作流** — 可以在活动设计完成后批量处理，不影响主流程
4. **PBL Box 订单有前置期约束** — 资源清单必须在开课前 2-4 周确定，这是一个硬截止日期
