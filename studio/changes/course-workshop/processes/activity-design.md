# Process Flow: 单个探究线索的活动设计流程

> Process name: 探究线索活动设计
> Trigger: 探究线索已确定（含关键问题、成功素养、学习目标）
> End states: 线索下所有活动（3-9个）设计完成 / 线索被调整需重新设计
> Boundary: 从线索拆分完成到活动编排定稿（不含资源匹配和双语翻译）
> Frequency: 每条线索 1 次，每套预案 3 条线索

## Flow Diagram

```
[Trigger: 探究线索已确定]
    │
    ├─ Input: 关键问题（如"果汁店要做什么口味的果汁？"）
    ├─ Input: 成功素养（如 ① 沟通能力 ② 思辨能力）
    ├─ Input: 学习目标（如 ① 能通过问卷调查了解偏好 ② 能综合应季水果确定果汁）
    └─ Input: 关键词列表
    │
    ▼
[确定活动数量和主题]
    │  ┌─────────────────────────┐
    │  │ 通常 3-5 个活动          │
    │  │ 每个活动 = 1 课时(20-30min)│
    │  │ 线索总时长 = 3-5 天       │
    │  └─────────────────────────┘
    ▼
[为每个活动依次设计]
    │
    ▼
┌──────── 循环: 对每个活动 ────────┐
│                                 │
│  [确定关键问题]                  │
│      │                          │
│      ▼                          │
│  [分配活动编码 PBL-C{x}-{y}]    │
│      │                          │
│      ▼                          │
│  [编写活动内容（3-4 步骤）]      │
│      │                          │
│      ▼                          │
│  ◇ 活动时长是否在 20-30 分钟？  │
│      ├─ 否（太长）→ [拆分为 2 个活动]│
│      ├─ 否（太短）→ [合并或扩展]  │
│      └─ 是 ↓                    │
│          [编写教师提示 Tips]      │
│              │                  │
│              ▼                  │

│                                 │

    │

[检查活动序列]
    │

```

## Decision Points

| ID | Decision | Condition | True path | False path |
|----|----------|-----------|-----------|------------|
| D1 | 活动时长 20-30 min？ | 根据步骤数和儿童注意力估算 | 写 Tips | 拆分或合并 |
| D2 | 活动间逻辑递进？ | 后一个活动依赖前一个活动的知识/经验 | 目标覆盖检查 | 调整顺序 |
| D3 | 覆盖学习目标？ | 每个学习目标至少被 1 个活动覆盖 | 完成 | 补充活动 |

## Actor Swim Lanes

```
课研主任:  [确定数量] → [设计活动内容] → [写 Tips] → [标注资源] → [检查序列]
课程专家:                    (review) ─────────────────────── [验证递进性和目标覆盖]
心理学家:                    (review) ─────────────────────── [检查年龄适配性]
```

## Key Observations

1. **活动设计是嵌套循环** — 每条线索 3-5 个活动，每套预案 3 条线索 = 9-15 个活动需要逐个设计
2. **时长估算是关键决策** — 高估导致活动太少，低估导致教师时间不够
3. **活动编码（PBL-C{x}-{y}）是强制规范** — 必须连续编号，线索号对应线索顺序
4. **教师提示是差异化价值** — 有经验的课研主任能写出好的 Tips，新手往往忽略
